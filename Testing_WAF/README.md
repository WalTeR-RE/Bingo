# Testing_WAF

Offline training and evaluation pipeline for Bingo's defensive WAF classifier.

## What this is

This folder builds the multiclass XGBoost model that the **Defensive Agent** uses at runtime to classify HTTP requests as benign or as a specific attack class. Training is done here, offline; the resulting model bundle is what `Ai-Agent/Defensive-Agent/waf_engine.py` loads from `waf_model.pkl`. Nothing in this folder runs during a live scan — it produces and validates the artifact the live engine consumes.

The pipeline trains on the **SR-BH 2020** dataset (CAPEC-labeled HTTP requests) and augments it with **SecLists** fuzzing payloads plus synthetic SSRF/RFI/SSTI/benign traffic, so the classifier covers attack classes that are sparse or absent in the base corpus. Features combine character-level TF-IDF n-grams with hand-built statistical and regex-category features (see `features.py`).

The dataset is **not** committed to the repo. Use `download_dataset.py` to fetch it.

## Files

| File | Responsibility |
| --- | --- |
| `features.py` | Feature extraction. `build_matrix(texts, tfidf)` stacks char TF-IDF with `stats_frame()` (length, special/digit/space ratios, Shannon entropy, path depth, param count, SQL-keyword density, and per-category regex counts/hit flags). `STAT_FEATURE_NAMES` and `REGEX_CATEGORIES` (sqli, xss, traversal, rce, rfi_ssrf, ssti) are defined here and exported into the model bundle. |
| `waf_train.py` | Baseline trainer. Loads SR-BH 2020 directly from the CSV, maps CAPEC columns to labels (`LABELS`), collapses/drops rare classes, subsamples per class, then fits the TF-IDF + XGBoost model. Also exposes `load()`, reused by `augment_and_train.py`. |
| `augment_and_train.py` | Augmented trainer (the one to use). Filters SR-BH to core classes, then adds SecLists payloads (XSS, SQLi, LFI, command injection) and synthetic SSRF/RFI/SSTI/benign requests wrapped in realistic GET/POST request templates, before training. Writes `waf.pkl` + `test_split.pkl`. |
| `evaluate.py` | Evaluates `waf.pkl` against the held-out `test_split.pkl`: prints multiclass accuracy / macro-F1 / classification report, and benchmarks the **new** model against the **currently deployed** engine (`Ai-Agent/Defensive-Agent/waf_engine.py` + `waf_model.pkl`) on an attack-vs-benign basis. Saves charts. |
| `download_dataset.py` | Fetches SR-BH 2020 (`data_capec_multilabel.csv`, ~436 MB) from Harvard Dataverse into `../dataset/`. Skips if already present. |
| `charts/` | Generated by `evaluate.py`: `new_model_confusion.png`, `model_comparison.png`. |

## Outputs

Both trainers write into this folder:

- `waf.pkl` — model bundle: `{model, vectorizer, label_encoder, stat_feature_names}`.
- `test_split.pkl` — held-out test set `{text, y}` for `evaluate.py`.

To deploy a newly trained model, copy `waf.pkl` to `Ai-Agent/Defensive-Agent/waf_model.pkl` (the runtime path `evaluate.py` compares against).

## Usage

Run from this folder (Python with `numpy`, `pandas`, `scipy`, `scikit-learn`, `xgboost`, `matplotlib`):

```bash
# 1. Fetch the dataset (~436 MB into ../dataset/)
python download_dataset.py

# 2a. Train the augmented model (recommended)
python augment_and_train.py

# 2b. ...or the baseline SR-BH-only model
python waf_train.py

# 3. Evaluate and compare against the deployed engine
python evaluate.py
```

### Paths and dependencies

- Dataset CSV is expected at `../dataset/data_capec_multilabel.csv`.
- `augment_and_train.py` reads SecLists fuzzing lists from `../SecLists/` (sibling of this folder). Missing lists are skipped with a warning; the synthetic payloads still run.
- `evaluate.py` requires both `waf.pkl` and `test_split.pkl` from a prior training run, and imports the live engine from `../Ai-Agent/Defensive-Agent/`.

### Environment variables (`waf_train.py`)

| Variable | Default | Effect |
| --- | --- | --- |
| `WAF_MAX_ROWS` | `0` (all) | Cap rows read from the CSV (quick smoke runs). |
| `WAF_CLASS_CAP` | `60000` | Per-class subsample cap before training. |
